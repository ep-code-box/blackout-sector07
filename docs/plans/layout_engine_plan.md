# 전역 레이아웃 엔진(Box Model) 도입 계획

## 1. 개요
현재 하드코딩된 좌표 방식의 한계를 극복하기 위해, 상용 엔진 스타일의 '컨테이너 기반 자동 배치 시스템'을 구축합니다. 이를 통해 해상도 변화와 요소 추가에 유연하게 대응합니다.

## 2. 주요 시스템 설계

### A. 컨테이너 & 스택 엔진 (`theme.lua`)
- **`UI.beginLayout(x, y, w, h, p)`**: 특정 UI 영역을 선언하고 내부 패딩을 설정합니다.
- **`UI.nextItem(h, m)`**: 다음 UI 요소(버튼, 게이지 등)가 그려질 위치를 자동으로 계산하고 Y축을 전진시킵니다.
- **`UI.splitRow(h, n, m)`**: 현재 행을 N개의 컬럼으로 균등 분할하여 반환합니다. (예: 3분할 스탯창 구현 시 사용)

### B. UI 컴포넌트 리팩토링
- **`StatusInfo.lua`**: 3분할 레이아웃을 `splitRow`를 사용하여 논리적으로 재구축합니다. (좌표 노가다 제거)
- **`hub_main.lua`**: 우측 패널들의 수직 배치를 컨테이너 시스템으로 통합하여 간격을 일정하게 유지합니다.
- **`combat.lua`**: 파티원 5인의 배치를 그리드 시스템으로 관리하여 가변 인원에 대응합니다.

## 3. 구현 단계
1. **`theme.lua` 업데이트**: 레이아웃 스택 및 자동 계산 함수군 추가.
2. **`StatusInfo.lua` 적용**: 가장 복잡한 3분할 창을 새 엔진으로 리팩토링하여 검증.
3. **전역 적용**: 나머지 모든 UI 화면에서 직접 좌표 지정을 제거하고 레이아웃 엔진 호출로 교체.

## 4. 기대 효과
- **유지보수성:** 요소 하나를 추가해도 아래 요소들의 좌표를 일일이 수정할 필요가 없습니다.
- **일관성:** 모든 화면에서 여백(Padding)과 간격(Margin)이 통일되어 상용 게임급 마감을 보여줍니다.
- **확장성:** 4:3, 16:9 등 어떤 해상도에서도 비율에 맞는 배치가 즉시 가능해집니다.
