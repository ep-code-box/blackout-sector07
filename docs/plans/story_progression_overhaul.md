# 메인 스토리 및 육성 체감 강화 계획

## 1. 개요
단순 전투 반복을 넘어, 챕터 기반의 메인 서사를 도입하고 플레이어가 획득한 스킬 포인트를 사용하여 스킬을 커스터마이징할 수 있는 시스템을 구축합니다.

## 2. 주요 개선 사항

### A. 챕터 기반 메인 스토리 엔진 (`StoryManager` & `data_story.lua`)
- **챕터 시스템:** 게임을 여러 챕터로 나누어, 특정 보스 처치나 퀘스트 완료 시 자동으로 메인 스토리 이벤트가 발생하도록 구현합니다.
- **서사적 보상:** 스토리 진행에 따라 새로운 용병이 합류하거나, 상점의 '금지된 기술(아이템)'이 해금되는 잠금 해제 시스템을 연동합니다.
- **진행 가이드:** 현재 메인 목표(예: "섹터 07의 심장부로 진입하세요")를 화면 상단이나 로그에 지속적으로 노출하여 동기를 부여합니다.

### B. 스킬 업그레이드 시스템 (`hub_skills.lua`)
- **스킬 트리:** 레벨업 시 지급되는 `skill_points`를 소모하여 기존 스킬의 위력을 높이거나, SP 소모량을 줄이는 '특성' 시스템을 도입합니다.
- **육성 차별화:** 같은 '사무라이' 클래스라도 공격력에 올인할지, 유틸리티(상태이상)에 투자할지 플레이어가 선택할 수 있게 합니다.

### C. 성장의 시각적 피드백
- **레벨업 연출:** 퀘스트 완료 시 화면 중앙에 화려한 글리치 효과와 함께 "LEVEL UP" 배너가 출력되도록 FX를 추가합니다.
- **성장 리포트:** 스탯을 올릴 때마다 "공격력 +5 상승" 등 구체적인 수치 변화를 즉시 시각화하여 '강해짐'을 체감하게 합니다.

## 3. 구현 단계
1. **`data_story.lua`**: 챕터별 트리거 조건과 대화 스크립트 작성.
2. **`StoryManager.lua`**: 챕터 상태 저장 및 자동 이벤트 트리거 로직 추가.
3. **`hub_skills.lua`**: 스킬 포인트 투자 UI 및 데이터 반영 로직 구현.
4. **`FXManager.lua`**: 레벨업 및 스토리 해금 전용 특수 연출 추가.

## 4. 검증 계획
- 보스 처치 후 즉시 다음 스토리 대화가 시작되는지 확인.
- 스킬 포인트 투자 후 실제 전투 대미지에 반영되는지 수치 대조.
- 세이브 후 다시 시작했을 때 진행 중인 챕터가 정확히 복구되는지 테스트.
