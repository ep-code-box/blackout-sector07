# 하이테크 스탯창 리뉴얼 및 운명 스토리 엔진 구축 계획

## 1. 개요
현재의 밋밋한 스탯창을 전문적인 진단 터미널 스타일로 개편하고, 플레이어의 선택이 게임의 물리적 상태(NPC 생존, 맵 해금, 퀘스트 분기)를 바꾸는 '운명 엔진'을 도입합니다.

## 2. 주요 개선 사항

### A. 스탯창 리뉴얼: "바이오메트릭 진단 터미널"
- **3구역 레이아웃:** 
    - **Core Info:** 이름, 직업, 레벨, 고퀄리티 초상화.
    - **Data Matrix:** 스탯뿐만 아니라 공격력, 방어력, 회피율 등 '전투 결과 수치'를 실시간으로 계산하여 표기.
    - **Perk Deck:** 현재 해금된 스탯 기반 특성(Perk)들을 별도 카드로 시각화.
- **애니메이션:** 스탯 투자 시 화면이 미세하게 지지직거리는 글리치 연출 추가.

### B. 운명 스토리 엔진 (Narrative Fate Engine)
- **World State 시스템:** `SaveManager`와 연동되는 전역 플래그(Flag) 시스템 구축. (예: `is_luna_betrayed`, `sector_07_core_exploded`)
- **결과 기반 액션:** 대화 선택지에 따라 단순히 텍스트만 나오는 것이 아니라 다음과 같은 액션 실행:
    - **`set_flag`:** 세계 상태 변경.
    - **`give_item`:** 선택 보상 지급.
    - **`unlock_map`:** 새로운 탐험 구역 해금.
    - **`trigger_combat`:** 대화 중 즉시 전투 돌입.
- **반응형 대화:** 특정 플래그가 활성화된 상태에서만 등장하는 대화문 지원.

## 3. 구현 단계
1. **`status_info.lua`**: 전문 터미널 레이아웃으로 전면 재작성.
2. **`data_story.lua`**: 플래그 상태와 액션이 포함된 확장 스크립트 구조로 변경.
3. **`StoryManager.lua`**: 선택지 클릭 시 `actions` 테이블을 해석하고 실행하는 로직 추가.
4. **`save_manager.lua`**: `WorldState` 플래그가 세이브 파일에 포함되도록 업데이트.

## 4. 검증 계획
- 스탯창에서 STR을 올렸을 때 'ATTACK_PWR' 수치가 즉시 갱신되는지 확인.
- 스토리 선택지에서 "아이템 받기"를 골랐을 때 실제로 인벤토리에 들어오는지 테스트.
- 특정 선택 후 세이브/로드를 해도 세계의 상태(Flag)가 유지되는지 확인.
